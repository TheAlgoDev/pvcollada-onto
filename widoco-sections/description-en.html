Design decisions

The ontology is serialised in Turtle (OWL 2 DL profile). All classes carry the MDS-Onto required annotation properties mds:hasDomain ("builtenv"), mds:hasSubDomain ("pvcollada"), and mds:hasStudyStage. Quantitative data properties are annotated with qudt:applicableUnit pointing to QUDT unit IRIs, so unit information is machine-readable without embedding it in property names.

Enumeration modelling

XSD simpleType enumerations (cell technology, rack type, inverter type, etc.) are modelled as owl:Class + owl:oneOf named individuals rather than as rdfs:Datatype restrictions. This preserves the ability to attach annotations and alignment links to each enumeration member (e.g. pvc:PERC skos:exactMatch mds:PassivatedEmitterRearCell).

Business rules

Two Schematron conditional-field rules from pvcollada_business_2.0.sch are re-expressed as OWL 2 subclass restrictions:

  • pvc:FixedTiltRack must have pvc:azimuth and pvc:tilt (minCardinality 1).
  • pvc:TrackerRack must have pvc:trackerAzimuth (minCardinality 1).

The two subclasses are declared owl:AllDisjointClasses. The MPPT URL uniqueness constraint (also in the Schematron) cannot be expressed in OWL 2 DL; it is documented as a comment and enforced via the SPARQL SHACL shapes file (shapes/pvcollada_shape.ttl).

MDS-Onto alignment summary

Four class bridges use rdfs:subClassOf: pvc:Module → mds:PhotovoltaicModule; pvc:Inverter → mds:Inverter; pvc:Rack → mds:Racking; pvc:Project → mds:PhotovoltaicSite. One enumeration member uses skos:exactMatch (pvc:PERC); two enumeration classes use skos:closeMatch; thirteen data properties use rdfs:seeAlso links to MDS-Onto measurement classes.
